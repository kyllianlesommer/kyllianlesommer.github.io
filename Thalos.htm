Nagios

logo de Nagios

Nagios est une application libre qui permet la surveillance système et du réseau. Elle surveille des hôtes et/ou des services spécifiés, alertant lorsque les systèmes ont des dysfonctionnements et quand ils repassent en fonctionnement normal.

Par la suite j’ai travaillé sur un logiciel nommé Nagios pour étudier le Monitoring grâce au protocole SNMP ( Simple Network Management Protocol )

Début sur Nagios, j’ai tout d’abord étudier comment marchais le protocole SNMP, c’est un protocole de Couche 7 du modèle OSI, il permet la remontée d’informations via des OID

WSL ( Windows Subsystem for Linux )

Windows Subsystem for Linux est un système de compatibilité entre les deux systèmes d’exploitation Windows/Linux, pour y exécuter des binaires comme natifs, est disponible sur Windows 10 et Windows Server 2019. il y a deux versions.

j’ai travaillé avec WSL sur un Windows server 2019, pour l’installer j’ai procéder ainsi:

Étape 1 préparation du système :

Lancement d’un PowerShell en tant qu’Administrateur

tapez la commande suivante pour installer WSL.

Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux

Étape 2 téléchargement de la distribution :

téléchargement des distributions via ce lien

si téléchargé un fichier .appx exemple ( Ubuntu_2004.2020.424.0_x64.appx )

il faut tout d’abord le mettre dans le dossier ou il sera utiliser pour faciliter l’utilisation

une fois bien mis, il faut le renommer, avec la commande

Rename-Item .\Ubuntu_2004.2020.424.0_x64.appx .\Ubuntu.zip

par la suite, il faut l’extraire

Expand-Archive .\Ubuntu.zip .\Ubuntu

En parallèle il faut créer une nouvelle variable d’environnement pour pouvoir lancer la commande wsl,

$userenv = System.Environment]::GetEnvironmentVariable("Path&quot;, &quot;User") [System.Environment]::SetEnvironmentVariable("PATH", $userenv + ";C:\WSL\Ubuntu", "User")

Étape 3 installer les distributions :

lancement de WSL :

Toujours dans le PowerShell en administrateur et dans le bon dossier, ( C:\WSL\Ubuntu\ )
lancez la commande .\ubuntu2004.exe qui elle va lancer la distribution Ubuntu pour la Première fois.

Quelques minutes vont s’écouler et elle va vous demander quel utilisateur voulez vous créer, par exemple
login : user
password : m4utD2p@5$e

Étape 4 paramétrage de la distribution :

Mise à jour des paquets ( sudo apt update )

Étape 5 accès distant (SSH) :

SSH ( Secure Shell )

Mise à jour du paquet ssh
sudo apt install ssh

modification du fichier /etc/ssh/sshd_config
sudo vi /etc/ssh/sshd_config

changement du port d’entrée en 22000 car celui de Windows est de base 22 donc pour aller sur WSL et non pas Windows en ssh il faut faire cela

modification lignes :
PermitRootLogin yes
PasswordAuthentication yes

création des clés ssh :
sudo ssh-keygen -A

Démarrage du service ssh :
sudo service ssh start

pour éviter d’entrer le mot de passe nous allons faire visudo.

pour cela, faire sudo visudo

résultat de la commande sudo visudo

modifier la ligne %sudo et mettre au lieu de ALL à la fin, mettre NOPASSWD:ALL.
